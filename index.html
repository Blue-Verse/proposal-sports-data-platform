프로젝트 소개

해외 스포츠 경기 데이터를 수집·가공하여 국내 사용자에게 직관적인 UI/UX로 정보를 제공하는 플랫폼을 구축합니다.

핵심 요구사항 분석

  • 해외 스포츠 데이터 취합 — 지정된 다수의 해외 스포츠 정보 사이트에서 경기 현황, 선수 정보(부상 등), 관련 기록 등 텍스트·수치 데이터 수집
  • 데이터 통합 및 가공 — 수집 데이터를 표준화된 포맷으로 가공하여 DB 저장, 사용자에게 일관된 형태로 제공
  • 스포츠 정보 제공 UI — 종목별, 리그별, 팀별, 선수별 분류·탐색·조회 기능
  • 관리자 페이지 — 데이터 소스(URL) 관리, 수집 현황 모니터링, 주요 데이터 관리 및 통계 확인
  • 반응형 웹 — PC Web + Mobile Web 반응형 지원
  • UI/UX 디자인 — 세련되고 직관적인 디자인으로 가독성·접근성 극대화

제안 포인트

1. 데이터 수집·파이프라인 전문성

다수의 외부 데이터 소스를 통합·가공하는 프로젝트를 다수 수행한 경험이 있습니다. Puppeteer/Playwright 기반 스크래핑 시스템과 스케줄링 파이프라인을 구축하여 안정적인 데이터 수집 환경을 제공합니다. 공식 API가 제공되는 경우 API 연동을 우선하고, 스크래핑은 API 미제공 소스에 한정하여 안정성을 확보합니다.

2. 대시보드·데이터 시각화 역량

50+ 페이지 규모의 관리자 대시보드, 통계·분석 화면, 데이터 테이블 구현 경험을 보유하고 있습니다. Recharts/Chart.js를 활용한 데이터 시각화와 TailwindCSS 기반의 반응형 UI를 통해 사용자 친화적인 정보 탐색 경험을 제공합니다.

3. 기획부터 배포까지 원스톱 진행

상세 기획(요구사항 정의, 기능 정의, 화면 설계), UI/UX 디자인(Figma), 프론트엔드·백엔드 개발, 서버·DB·인프라 구성까지 전 과정을 일관된 품질로 진행합니다. 기획 상태가 '간단히 정리한 상태'이므로, 요구사항 구체화부터 함께 진행하겠습니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

NBA 등 해외에서 진행되는 다양한 스포츠 경기의 관련 정보가 여러 해외 사이트에 분산되어 있어, 국내 사용자가 이를 통합적으로 확인하기 어려운 상황입니다. 본 프로젝트는 이러한 분산된 경기 현황, 선수 정보, 분석 데이터를 취합하고, 사용자 친화적인 UI/UX 디자인을 적용하여 정보의 가독성과 접근성을 극대화한 정보 플랫폼을 구축하는 것을 목표로 합니다.

2. 사용자 정의

사용자 유형설명주요 기능
일반 사용자해외 스포츠 경기 정보를 탐색하는 국내 사용자종목/리그/팀/선수별 정보 조회, 경기 현황 확인, 검색·필터링
관리자플랫폼 운영 및 데이터 관리 담당자데이터 소스(URL) 관리, 수집 현황 모니터링, 데이터 관리, 통계 확인

3. 기능 요구사항

3.1 해외 스포츠 데이터 취합

  • [필수] 지정된 다수의 해외 스포츠 정보 사이트에서 데이터 수집
  • [필수] 경기 현황 (일정, 결과, 스코어) 수집
  • [필수] 선수 정보 (프로필, 부상 현황, 통계) 수집
  • [필수] 관련 기록 및 분석 데이터 수집
  • [필수] 공식 API 우선 연동, 미제공 소스는 스크래핑
  • [선택] 수집 주기 설정 (실시간/주기적)

3.2 데이터 통합 및 가공

  • [필수] 수집 데이터 표준화 포맷 정의 및 가공
  • [필수] 가공된 데이터 DB 저장
  • [필수] 사용자에게 일관된 형태로 표시
  • [선택] 데이터 품질 검증 (중복 제거, 누락 감지)

3.3 스포츠 정보 제공 UI

  • [필수] 종목별 분류 및 탐색
  • [필수] 리그별 분류 및 탐색
  • [필수] 팀별 분류 및 상세 정보
  • [필수] 선수별 분류 및 상세 정보
  • [필수] 경기 현황 목록 및 상세
  • [필수] 검색 및 필터링 기능
  • [필수] PC/Mobile 반응형 지원

3.4 관리자 페이지

  • [필수] 데이터 소스(URL) CRUD 관리
  • [필수] 수집 현황 모니터링 (성공/실패/마지막 수집 시간)
  • [필수] 주요 데이터 관리 (종목, 리그, 팀, 선수 CRUD)
  • [필수] 통계 확인 (수집 건수, 사용자 접근 통계)
  • [선택] 수집 스케줄 관리

5. 기술 제약사항

  • 해외 스포츠 데이터 소스의 접근성 및 이용약관에 따라 수집 방식이 달라질 수 있음
  • 스크래핑 대상 사이트의 구조 변경 시 파서 업데이트 필요
  • 실시간 데이터의 경우 소스 사이트의 업데이트 주기에 의존
  • 해외 사이트 접근 시 네트워크 레이턴시 고려 필요

6. 범위 외 사항 (Out of Scope)

  • 네이티브 모바일 앱 (iOS/Android) 개발
  • 실시간 경기 중계 기능
  • 사용자 간 커뮤니티/댓글 기능 (1차 범위 외)
  • 결제/구독 시스템
  • 다국어 지원 (1차는 한국어 전용)
